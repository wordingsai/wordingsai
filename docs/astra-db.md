# Astra DB (DataStax) — Vector + AI payload storage

Wordings.ai can use **Astra DB Serverless** as the vector database. Chunking stays in the app (`splitText`); **embeddings are generated by Astra** via `$vectorize` when documents are inserted.

Neon Postgres remains the source of truth for users, contracts, clauses, and relational data. Large AI outputs are duplicated into Astra for search and offload.

## Environment variables

```env
USE_ASTRA_VECTOR=true
ASTRA_DB_APPLICATION_TOKEN=your_astra_token
ASTRA_DB_API_ENDPOINT=https://....apps.astra.datastax.com

# After migration: new clauses go to Astra only (default true when Astra is on)
ASTRA_SKIP_NEON_CHUNK_MIRROR=true

# Optional collection names (defaults shown)
ASTRA_COLLECTION_CLAUSE_CHUNKS=clause_chunks
ASTRA_COLLECTION_CONTRACT_CHUNKS=contract_chunks
ASTRA_COLLECTION_AI_GENERATIONS=ai_generations
ASTRA_COLLECTION_WAR_EXCLUSIONS=war_exclusions

# Optional: max cosine distance for matches (default 0.6)
ASTRA_VECTOR_MAX_DISTANCE=0.6

# NVIDIA vectorize max 512 tokens — chunk size (~720 chars default)
ASTRA_VECTORIZE_MAX_CHARS=720
```

Install the client (if not already):

```bash
pnpm add @datastax/astra-db-ts
```

## Create collections in Astra

For each collection, enable **Vectorize** on the **`content`** field in the Astra UI (or Data API), using your preferred embedding provider (e.g. OpenAI, NVIDIA on Astra, Voyage).

Recommended collections:

| Collection        | Purpose                         | Key metadata fields                                                                          |
| ----------------- | ------------------------------- | -------------------------------------------------------------------------------------------- |
| `clause_chunks`   | Clause library semantic search  | `clauseId`, `organizationId`, `workspaceId`, `isGlobal`, `clauseName`, `library`, `category` |
| `contract_chunks` | Contract RAG                    | `contractId`, `contractVersionId`, `content`                                                 |
| `ai_generations`  | Large AI JSON (maps, summaries) | `contractId`, `kind`, `payload`, `content`                                                   |
| `war_exclusions`  | War exclusion matching          | `organizationId`, `title`, `clauseText`                                                      |

Documents are inserted with:

```json
{
  "content": "text to embed",
  "$vectorize": "same text — Astra embeds automatically"
}
```

## What runs on Astra vs Postgres

| Feature                         | Astra                  | Postgres                                 |
| ------------------------------- | ---------------------- | ---------------------------------------- |
| Clause chunk vectors            | Yes (`$vectorize`)     | Optional mirror rows without `embedding` |
| Contract RAG chunks             | Yes                    | Optional mirror rows                     |
| Document map / summary payloads | Yes (`ai_generations`) | `structured_content`, `analysis` jsonb   |
| Auth, billing, rules            | No                     | Yes                                      |

## Code layout

- `src/lib/astra/config.ts` — env + feature flag
- `src/lib/astra/client.ts` — Data API client
- `src/lib/astra/vector-store.ts` — insert/search/delete
- `src/lib/vector/index.ts` — thin routing facade

When `USE_ASTRA_VECTOR` is off (or credentials missing), the app uses **PGVector** and `src/lib/embedding.ts` as before.

## Migrating existing Postgres embeddings

You do **not** copy old 1024-dim vectors from Postgres. Astra **re-embeds** each document via `$vectorize` (NVIDIA 1024 is correct).

### Option A — Direct import (recommended)

```bash
# Clause library → clause_chunks collection
pnpm astra:import-clauses

# Contract RAG chunks → contract_chunks collection
pnpm astra:migrate-contracts
```

### Option B — JSON export, then import

Use this if you want to inspect data or load via another tool.

```bash
# 1. Export Astra-shaped JSON from Postgres
pnpm astra:export-clauses
# → data/astra-clause-chunks-export.json

# 2. Import into Astra (same shape; triggers vectorize)
pnpm astra:import-clauses -- --from-json=data/astra-clause-chunks-export.json
```

Each document must include **`content`** and **`$vectorize`** (same string). Example:

```json
{
  "_id": "uuid",
  "docType": "clause_chunk",
  "clauseId": "...",
  "organizationId": "...",
  "workspaceId": null,
  "isGlobal": false,
  "clauseName": "Termination",
  "library": "LMA",
  "category": "Termination",
  "content": "text to embed...",
  "$vectorize": "text to embed...",
  "createdAt": "2026-05-20T12:00:00.000Z"
}
```

Manual upload in the Astra UI only works if the loader sends `$vectorize` on insert — otherwise embeddings will be missing.

## Re-sync after enabling Astra

1. Set env vars and create vectorize-enabled collections.
2. Re-run clause sync: trigger `clause/sync` per clause or run `embedSingleClause` via Inngest.
3. Re-run contract analysis (force) to rebuild `contract_chunks` and `ai_generations`.

## After clause migration — clean Neon

Once `pnpm astra:import-clauses` succeeds:

```bash
# Preview
npx tsx scripts/cleanup-neon-vectors.ts --clauses --dry-run

# Remove vectors from Neon (recommended: null embeddings, keep rows)
pnpm astra:cleanup-neon

# Or delete all clause_chunks rows entirely
npx tsx scripts/cleanup-neon-vectors.ts --clauses --delete-rows
```

With `USE_ASTRA_VECTOR=true`, **new/updated clauses** sync to Astra only (`embedSingleClause` → Inngest `clause/sync`). Neon `clause_chunks` mirror is skipped by default.

## War exclusions migration

```bash
pnpm astra:migrate-war
npx tsx scripts/cleanup-neon-vectors.ts --war
```

Rule evaluation (`findWarExclusionMatches`) and deep rules use Astra when `USE_ASTRA_VECTOR=true`, with full metadata loaded from Postgres.

## AI generations (auto during analysis)

| Collection        | When it fills                                                                     |
| ----------------- | --------------------------------------------------------------------------------- |
| `ai_generations`  | Fast analysis saves document map + summary (`structured_content`, `fast_summary`) |
| `contract_chunks` | Deep analysis / `mainAnalysisJob` only (`pnpm astra:migrate-contracts`)           |

## Client setup note (SDK v2)

```ts
const client = new DataAPIClient(TOKEN);
const db = client.db(API_ENDPOINT, { token: TOKEN });
```

Endpoint must end with `.astra.datastax.com` (not `.om`).

## Hybrid mode (legacy)

If `ASTRA_SKIP_NEON_CHUNK_MIRROR=false`, chunk text is mirrored in Postgres without embeddings. Default when Astra is on: **Astra only** for clause vectors.
